This guide will take you from installation to a working authenticated endpoint. By the end, you'll have:

- A configured `AuthModule` in your NestJS application
- A protected route that requires authentication
- A public route accessible without authentication

<Note>
  This guide assumes you've already [installed](/installation) the library and configured Better Auth. If not, complete those steps first.
</Note>

## Setup Steps

<Steps>

<Step title="Disable NestJS Body Parser">

Better Auth needs to handle the raw request body for authentication routes. Disable NestJS's built-in body parser in your `main.ts` file:

```typescript main.ts
import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule, {
    // Disable built-in body parser
    bodyParser: false,
  });
  await app.listen(process.env.PORT ?? 3000);
}
bootstrap();
```

<Warning>
  Don't worry! The library will re-add body parsers for non-auth routes automatically. Your regular routes will still receive parsed JSON and form data.
</Warning>

</Step>

<Step title="Import AuthModule in Your Root Module">

Import the `AuthModule` in your root module (typically `app.module.ts`) and provide your Better Auth instance:

```typescript app.module.ts
import { Module } from '@nestjs/common';
import { AuthModule } from '@thallesp/nestjs-better-auth';
import { auth } from './auth'; // Your Better Auth instance
import { AppController } from './app.controller';

@Module({
  imports: [
    AuthModule.forRoot({
      auth,
      bodyParser: {
        json: { limit: '2mb' },
        urlencoded: { limit: '2mb', extended: true },
      },
    }),
  ],
  controllers: [AppController],
})
export class AppModule {}
```

<Info>
  The `bodyParser` configuration re-adds JSON and URL-encoded parsers for your regular routes. You can customize limits and options as needed.
</Info>

</Step>

<Step title="Create Your First Protected Route">

By default, **all routes are protected** with the global `AuthGuard`. Create a controller with a protected route that requires authentication:

```typescript app.controller.ts
import { Controller, Get } from '@nestjs/common';
import { Session, UserSession } from '@thallesp/nestjs-better-auth';

@Controller('users')
export class AppController {
  @Get('me')
  async getProfile(@Session() session: UserSession) {
    return {
      message: 'This route requires authentication',
      user: session.user,
      session: session.session,
    };
  }
}
```

The `@Session()` decorator extracts the authenticated user's session from the request. This route will return a `401 Unauthorized` error if the user is not authenticated.

</Step>

<Step title="Create a Public Route">

To create a route that doesn't require authentication, use the `@AllowAnonymous()` decorator:

```typescript app.controller.ts
import { Controller, Get } from '@nestjs/common';
import { AllowAnonymous, Session, UserSession } from '@thallesp/nestjs-better-auth';

@Controller('users')
export class AppController {
  @Get('me')
  async getProfile(@Session() session: UserSession) {
    return {
      message: 'This route requires authentication',
      user: session.user,
      session: session.session,
    };
  }

  @AllowAnonymous()
  @Get('public')
  async publicRoute() {
    return {
      message: 'This route is public and accessible without authentication',
    };
  }
}
```

</Step>

<Step title="Test Your Routes">

Start your NestJS application:

```bash
npm run start:dev
```

Your Better Auth routes are now available at `/api/auth/*` (or the `basePath` you configured in Better Auth).

**Test the public route** (should work immediately):

```bash
curl http://localhost:3000/users/public
```

**Test the protected route** (should return 401 Unauthorized):

```bash
curl http://localhost:3000/users/me
```

**Sign up a user** using Better Auth:

```bash
curl -X POST http://localhost:3000/api/auth/sign-up/email \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "securepassword123",
    "name": "Test User"
  }'
```

**Sign in and get a session token**:

```bash
curl -X POST http://localhost:3000/api/auth/sign-in/email \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "securepassword123"
  }' \
  -c cookies.txt
```

**Access the protected route with the session cookie**:

```bash
curl http://localhost:3000/users/me \
  -b cookies.txt
```

You should now see the authenticated user's data!

</Step>

</Steps>

## Optional Authentication

Sometimes you want a route that works for both authenticated and unauthenticated users. Use the `@OptionalAuth()` decorator:

```typescript
import { Controller, Get } from '@nestjs/common';
import { OptionalAuth, Session, UserSession } from '@thallesp/nestjs-better-auth';

@Controller('users')
export class AppController {
  @OptionalAuth()
  @Get('profile')
  async getProfile(@Session() session: UserSession | undefined) {
    if (session?.user) {
      return {
        authenticated: true,
        user: session.user,
      };
    }
    
    return {
      authenticated: false,
      message: 'You are not logged in',
    };
  }
}
```

<Info>
  With `@OptionalAuth()`, the `session` parameter can be `undefined` if the user is not authenticated. Always check if the session exists before accessing its properties.
</Info>

## Complete Example

Here's a complete working example with all three types of routes:

```typescript app.controller.ts
import { Controller, Get } from '@nestjs/common';
import {
  AllowAnonymous,
  OptionalAuth,
  Session,
  UserSession,
} from '@thallesp/nestjs-better-auth';

@Controller('test')
export class AppController {
  // Protected route - requires authentication
  @Get('protected')
  async protectedRoute(@Session() session: UserSession) {
    return {
      message: 'You are authenticated!',
      user: session.user,
    };
  }

  // Public route - no authentication required
  @AllowAnonymous()
  @Get('public')
  async publicRoute() {
    return {
      message: 'This route is public',
    };
  }

  // Optional authentication - works for both
  @OptionalAuth()
  @Get('optional')
  async optionalRoute(@Session() session: UserSession | undefined) {
    return {
      authenticated: !!session?.user,
      user: session?.user ?? null,
    };
  }
}
```
