This guide covers the complete setup for using NestJS Better Auth with the Fastify adapter, including Fastify-specific considerations for body parsing and CORS.

## Prerequisites

- A working NestJS application using Fastify
- Better Auth >= 1.5.0 installed and configured
- `@nestjs/platform-fastify` installed

## Installation

```bash
npm install @thallesp/nestjs-better-auth @nestjs/platform-fastify
```

### Optional Dependencies

For URL-encoded body parsing with nested objects:

```bash
npm install qs
```

<Note>
  The `qs` package is required only if you use `bodyParser.urlencoded` with `extended: true`. Without it, URL-encoded parsing falls back to simple parsing without nested object support.
</Note>

## Step 1: Configure Better Auth

Create your Better Auth instance:

```typescript title="auth.ts"
import { betterAuth } from "better-auth";

export const auth = betterAuth({
  basePath: "/api/auth",
  database: {
    // Your database configuration
  },
  emailAndPassword: {
    enabled: true,
  },
  trustedOrigins: ["http://localhost:3000"],
});
```

## Step 2: Configure Fastify Adapter

Create your NestJS application with the Fastify adapter:

```typescript title="main.ts"
import { NestFactory } from "@nestjs/core";
import { FastifyAdapter, NestFastifyApplication } from "@nestjs/platform-fastify";
import { AppModule } from "./app.module";

async function bootstrap() {
  const app = await NestFactory.create<NestFastifyApplication>(
    AppModule,
    new FastifyAdapter(),
    {
      // Disable NestJS's built-in body parser
      bodyParser: false,
    },
  );

  await app.listen(process.env.PORT ?? 3333, "0.0.0.0");
}
bootstrap();
```

<Warning>
  **Important:** Since we disable NestJS's built-in body parser, the `rawBody: true` option in `NestFactory.create()` has no effect. If you need access to `req.rawBody` (e.g., for webhook signature verification), use `bodyParser.rawBody` in `AuthModule.forRoot()` instead.
</Warning>

## Step 3: Import AuthModule

Import the `AuthModule` in your root module:

```typescript title="app.module.ts"
import { Module } from "@nestjs/common";
import { AuthModule } from "@thallesp/nestjs-better-auth";
import { auth } from "./auth";

@Module({
  imports: [
    AuthModule.forRoot({
      auth,
      bodyParser: {
        json: { limit: "2mb" },
        urlencoded: { limit: "2mb", extended: true },
        rawBody: true,
      },
    }),
  ],
})
export class AppModule {}
```

## Body Parser Configuration

On Fastify, the library registers custom body parsers using Fastify's native `useBodyParser` API.

### Default Configuration

```typescript
AuthModule.forRoot({
  auth,
  bodyParser: {
    json: {
      enabled: true,
      limit: "100kb",
      strict: true,
    },
    urlencoded: {
      enabled: true,
      extended: true,
      limit: "100kb",
    },
    rawBody: false,
  },
})
```

### URL-Encoded Parser with qs

When `extended: true` is set, the library uses the `qs` package for nested object parsing:

```typescript
AuthModule.forRoot({
  auth,
  bodyParser: {
    urlencoded: {
      enabled: true,
      extended: true, // Requires 'qs' package
      parameterLimit: 1000,
      depth: 5,
    },
  },
})
```

<Warning>
  If `qs` is not installed when `extended: true`, you'll get an error:
  
  ```
  Fastify bodyParser.urlencoded with extended: true requires the optional
  peer dependency 'qs'. Install 'qs' in your application to enable nested
  URL-encoded parsing.
  ```
</Warning>

### Simple URL-Encoded Parser

Without `qs`, use simple parsing:

```typescript
AuthModule.forRoot({
  auth,
  bodyParser: {
    urlencoded: {
      enabled: true,
      extended: false, // Uses URLSearchParams (no nested objects)
    },
  },
})
```

### Custom Body Size Limits

Fastify accepts size limits as numbers (bytes) or strings:

```typescript
AuthModule.forRoot({
  auth,
  bodyParser: {
    json: { limit: "5mb" },     // String format
    urlencoded: { limit: 5242880 }, // Number format (5MB in bytes)
  },
})
```

Supported string formats: `"2mb"`, `"500kb"`, `"1gb"`, `"1024b"`

### Enable Raw Body

For webhook signature verification:

```typescript
AuthModule.forRoot({
  auth,
  bodyParser: {
    rawBody: true,
  },
})
```

Access the raw body in your controllers:

```typescript
import { Controller, Post, Request } from "@nestjs/common";
import type { FastifyRequest } from "fastify";

@Controller("webhooks")
export class WebhookController {
  @Post("stripe")
  async handleStripeWebhook(@Request() req: FastifyRequest) {
    const signature = req.headers['stripe-signature'];
    const rawBody = (req as any).rawBody; // Buffer
    // Verify signature with raw body
  }
}
```

## CORS Configuration

Fastify CORS requires special handling because Better Auth routes are mounted through middleware.

### Automatic CORS (Recommended)

If you configure `trustedOrigins` in your Better Auth config, the module automatically applies CORS:

```typescript title="auth.ts"
export const auth = betterAuth({
  basePath: "/api/auth",
  trustedOrigins: [
    "http://localhost:3000",
    "https://myapp.com",
  ],
});
```

The module handles CORS for Better Auth routes automatically. No additional configuration needed.

### With @fastify/cors

If you already have `@fastify/cors` registered, the module detects it and skips duplicate registration:

```typescript title="main.ts"
import fastifyCors from "@fastify/cors";

async function bootstrap() {
  const app = await NestFactory.create<NestFastifyApplication>(
    AppModule,
    new FastifyAdapter(),
    { bodyParser: false },
  );

  // Register @fastify/cors for your app routes
  await app.register(fastifyCors, {
    origin: ["http://localhost:3000"],
    credentials: true,
  });

  await app.listen(3333, "0.0.0.0");
}
```

<Note>
  The module logs a warning when it detects `@fastify/cors` is already registered. Better Auth routes will still apply CORS from `trustedOrigins`. This is safe and expected behavior.
</Note>

### Disable Automatic CORS

If you want to fully manage Better Auth route CORS yourself:

```typescript
AuthModule.forRoot({
  auth,
  disableTrustedOriginsCors: true,
})
```

<Warning>
  Only set `disableTrustedOriginsCors: true` if you're implementing custom CORS handling for Better Auth routes. Otherwise, auth requests from your frontend may fail due to CORS errors.
</Warning>

## Fastify-Specific Considerations

### Body Parser Registration

On Fastify, the library:
1. Removes default content type parsers for `application/json` and `application/x-www-form-urlencoded`
2. Registers custom parsers using `useBodyParser`
3. Handles raw body attachment when `rawBody: true`

### Content Type Matchers

You cannot use function-based type matchers on Fastify:

```typescript
// ❌ This will throw an error on Fastify
bodyParser: {
  json: {
    type: (req) => req.headers['content-type'] === 'application/json',
  },
}

// ✅ Use string or string array instead
bodyParser: {
  json: {
    type: ["application/json", "application/vnd.api+json"],
  },
}
```

### Request and Response Types

Use Fastify types in your controllers:

```typescript
import type { FastifyRequest, FastifyReply } from "fastify";

@Controller("users")
export class UserController {
  @Get("me")
  async getProfile(@Request() req: FastifyRequest) {
    return req.session;
  }
}
```

### Middleware Integration

Custom middleware receives Express-style parameters but works with Fastify:

```typescript
import type { Request, Response, NextFunction } from "express";

AuthModule.forRoot({
  auth,
  middleware: (req: Request, res: Response, next: NextFunction) => {
    // This works on Fastify through adapter
    console.log(`Auth request: ${req.method} ${req.url}`);
    next();
  },
})
```

## Complete Example

Here's a complete working example with Fastify:

```typescript title="main.ts"
import { NestFactory } from "@nestjs/core";
import { FastifyAdapter, NestFastifyApplication } from "@nestjs/platform-fastify";
import { AppModule } from "./app.module";

async function bootstrap() {
  const app = await NestFactory.create<NestFastifyApplication>(
    AppModule,
    new FastifyAdapter(),
    {
      bodyParser: false,
    },
  );

  await app.listen(3333, "0.0.0.0");
}
bootstrap();
```

```typescript title="auth.ts"
import { betterAuth } from "better-auth";
import { prismaAdapter } from "better-auth/adapters/prisma";
import { prisma } from "./prisma";

export const auth = betterAuth({
  basePath: "/api/auth",
  database: prismaAdapter(prisma, {
    provider: "postgresql",
  }),
  emailAndPassword: {
    enabled: true,
  },
  trustedOrigins: ["http://localhost:3000"],
});
```

```typescript title="app.module.ts"
import { Module } from "@nestjs/common";
import { AuthModule } from "@thallesp/nestjs-better-auth";
import { auth } from "./auth";
import { UserController } from "./user.controller";

@Module({
  imports: [
    AuthModule.forRoot({
      auth,
      bodyParser: {
        json: { limit: "2mb" },
        urlencoded: { limit: "2mb", extended: true },
        rawBody: true,
      },
    }),
  ],
  controllers: [UserController],
})
export class AppModule {}
```

```typescript title="user.controller.ts"
import { Controller, Get } from "@nestjs/common";
import { Session, UserSession } from "@thallesp/nestjs-better-auth";

@Controller("users")
export class UserController {
  @Get("me")
  async getProfile(@Session() session: UserSession) {
    return session;
  }
}
```

## Troubleshooting

### Body Not Parsed

If request bodies are not being parsed:

1. Verify you've set `bodyParser: false` in `NestFactory.create()`
2. Check that body parsers are enabled in `AuthModule.forRoot()`
3. Ensure the Content-Type header matches the parser type

### qs Module Error

If you get an error about missing `qs`:

```bash
npm install qs
```

Or disable extended parsing:

```typescript
bodyParser: {
  urlencoded: { extended: false },
}
```

### CORS Issues on Fastify

If CORS is not working:

1. Verify `trustedOrigins` is set in your Better Auth config as an array (not a function)
2. Check that `disableTrustedOriginsCors` is not set to `true`
3. Ensure the origin header matches one of your trusted origins
4. If using `@fastify/cors`, verify it's registered before `app.listen()`

<Warning>
  Function-based `trustedOrigins` are not supported on Fastify unless you set `disableTrustedOriginsCors: true` and handle CORS manually.
</Warning>

### Raw Body Not Available

If `req.rawBody` is undefined:

1. Set `bodyParser.rawBody: true` in `AuthModule.forRoot()`
2. Do not use `rawBody: true` in `NestFactory.create()` (it has no effect)
