## What is NestJS Better Auth?

NestJS Better Auth is a comprehensive integration library that brings [Better Auth](https://www.better-auth.com/) to NestJS applications. It provides a seamless authentication and authorization experience with full support for NestJS's dependency injection, decorators, and modular architecture.

## Why NestJS Better Auth?

Better Auth is a powerful authentication framework, but integrating it with NestJS requires careful handling of middleware, request parsing, guards, and session management. This library solves all integration challenges and provides:

## Key Features

### Session Management

Access authenticated user sessions in your controllers, resolvers, and gateways:

```typescript
@Controller('users')
export class UserController {
  @Get('me')
  async getProfile(@Session() session: UserSession) {
    return session.user;
  }
}
```

### Role-Based Access Control

Protect routes with system-level roles (`@Roles()`) or organization-level roles (`@OrgRoles()`):

```typescript
@Roles(['admin'])
@Get('admin/dashboard')
adminDashboard() {
  return { message: 'System admin only' };
}

@OrgRoles(['owner', 'admin'])
@Get('org/settings')
orgSettings() {
  return { message: 'Organization owners and admins only' };
}
```

### Permission-Based Access Control

Implement fine-grained access control with Better Auth's access control system:

```typescript
@UserHasPermission({ permission: { project: ['create', 'update'] } })
@Post('projects')
createProject() {
  return { message: 'Project created' };
}
```

### Hook System

Create Better Auth hooks as NestJS services with dependency injection:

```typescript
@Hook()
@Injectable()
export class SignUpHook {
  constructor(private readonly emailService: EmailService) {}

  @BeforeHook('/sign-up/email')
  async handle(ctx: AuthHookContext) {
    await this.emailService.validateDomain(ctx.body.email);
  }
}
```

## Prerequisites

Before using this library, ensure you have:

- **NestJS** application (v11.1.6 or higher)
- **Better Auth** installed and configured (v1.5.0 or higher)
- **Node.js** 22.22.1 or higher

<Warning>
  Better Auth versions below 1.5.0 are deprecated and unsupported. Always use the latest version.
</Warning>
