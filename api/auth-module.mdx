## Overview

`AuthModule` is the core module that integrates Better Auth into your NestJS application. It provides authentication middleware, hooks support, global guard registration, and exception handling.

## Installation

```typescript
import { Module } from '@nestjs/common';
import { AuthModule } from '@better-auth/nestjs';
import { auth } from './auth';

@Module({
  imports: [
    AuthModule.forRoot({
      auth,
    }),
  ],
})
export class AppModule {}
```

#### Example: Basic Setup

```typescript
import { AuthModule } from '@better-auth/nestjs';
import { auth } from './auth';

@Module({
  imports: [
    AuthModule.forRoot({
      auth,
    }),
  ],
})
export class AppModule {}
```

#### Example: Custom Configuration

```typescript
AuthModule.forRoot({
  auth,
  bodyParser: {
    json: { limit: '10mb' },
    rawBody: true, // Enable for webhooks
  },
  disableTrustedOriginsCors: false,
}, {
  isGlobal: true,
  disableGlobalAuthGuard: false,
})
```

#### Example: Disable Global Guard

```typescript
// If you want to manually apply the guard to specific routes
AuthModule.forRoot({ auth }, {
  disableGlobalAuthGuard: true,
})
```

### forRootAsync()

Registers the AuthModule asynchronously, useful for dynamic configuration.

    <ParamField path="disableGlobalAuthGuard" type="boolean">

    </ParamField>
    
    <ParamField path="disableControllers" type="boolean">

    </ParamField>
  </Expandable>

#### Example: Async Configuration

```typescript
import { ConfigService } from '@nestjs/config';

@Module({
  imports: [
    AuthModule.forRootAsync({
      imports: [ConfigModule],
      inject: [ConfigService],
      useFactory: async (config: ConfigService) => ({
        auth: betterAuth({
          database: config.get('database'),
          // ... other options
        }),
      }),
    }),
  ],
})
export class AppModule {}
```

## Features

### Automatic Base Path Configuration

The module automatically:
- Normalizes the `basePath` from your Better Auth config (default: `/api/auth`)
- Excludes it from NestJS global prefix to avoid conflicts
- Sets up middleware to handle all Better Auth routes

### Hook Support

The module discovers and registers providers decorated with `@Hook()` that contain `@BeforeHook()` or `@AfterHook()` methods:

```typescript
import { Hook, BeforeHook } from '@better-auth/nestjs';
import type { AuthHookContext } from '@better-auth/nestjs';

@Hook()
@Injectable()
export class AuthHooks {
  @BeforeHook('/sign-in/email')
  async beforeSignIn(ctx: AuthHookContext) {
    console.log('User signing in:', ctx.body.email);
  }
}
```

### Global AuthGuard

By default, the module registers `AuthGuard` globally. All routes require authentication unless marked with:
- `@AllowAnonymous()` - Allow unauthenticated access
- `@OptionalAuth()` - Allow optional authentication

### CORS Configuration

Automatically configures CORS based on your Better Auth `trustedOrigins` setting. Supports both Express and Fastify adapters.

### Body Parser Management

Handles body parsing for auth routes with configurable options for JSON, URL-encoded, and raw body support.
